Uitvoeringsinformatie voor een taak.

`canonicalUrl` is altijd aanwezig als fallback — bruikbaar als het
portaal niet lokaal kan renderen.

`type` geeft het type van de definitie aan. Het portaal leidt
hier het CTA-label uit af (bijv. `betaling` → „Betalen”,
`upload` → „Document uploaden”). Onbekend of afwezig → generiek
label „Uitvoeren” + `canonicalUrl` als fallback.

`definitie` bevat de declaratieve definitie voor lokale uitvoering.
Alleen aanwezig in de detailrespons (`GET /taken/{uuid}`), en alleen
als de provider deze meelevert. Betaling heeft geen `definitie` —
betaaltaken leiden altijd tot een redirect via `canonicalUrl`.
Portalen die het type niet herkennen negeren `definitie` en gebruiken
`canonicalUrl` als fallback.

| `type` herkend | `definitie` aanwezig | Actie                                                                 |
|---|---|---|
| nee            | n.v.t.               | `canonicalUrl` → redirect; portaal voegt `returnUrl` toe              |
| ja             | nee                  | Type-specifieke CTA-knop; klik → `canonicalUrl` + `returnUrl`         |
| ja             | ja                   | Lokale uitvoering renderen; submit → `definitie.endpoint`             |
