# Use cases

Een **use case** beschrijft één samenhangende systeemfunctie vanuit het
perspectief van een actor (meestal de klant). Een use case kruist doorgaans
meerdere [schermen](../schermen/index.md) en verwijst naar specifieke
interactie-IDs op die schermen.

## Structuur per use case

Elke use case documenteert:

- **Actor** — wie voert het uit
- **Doel** — wat wil de actor bereiken
- **Voorwaarden** — wat moet al waar zijn
- **Hoofdflow** — genummerde stappen, elk verwijzend naar een interactie-ID
- **Alternatieve flows** — afwijkingen en foutscenario's
- **Resultaat** — de toestand na afloop
